# Code of Conduct

## Our Commitment

We are committed to creating an open, inclusive, and respectful environment for everyone.  
This project is a space where people can collaborate, learn, and build without fear of harassment or discrimination.

We expect all participants—contributors, maintainers, and users—to treat each other with respect and professionalism.

---

## Expected Behavior

When participating in this project, you agree to:

- Be respectful and considerate in communication
- Welcome constructive feedback and different perspectives
- Focus on what is best for the community and the project
- Show empathy toward other contributors
- Use inclusive and professional language

---

## Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment, intimidation, or discrimination of any kind
- Offensive comments related to identity (e.g. race, gender, religion, disability)
- Personal attacks, insults, or trolling
- Public or private harassment
- Publishing others’ private information without consent
- Any conduct that could reasonably be considered inappropriate in a professional setting

---

## Scope

This Code of Conduct applies within all project spaces, including:

- GitHub repositories (issues, pull requests, discussions)
- Community forums or chat platforms
- Social media interactions related to the project
- Any other spaces officially associated with the project

---

## Enforcement

Project maintainers are responsible for enforcing this Code of Conduct.

They may take any action deemed appropriate, including:

- Warning the offender
- Removing or editing content
- Temporarily or permanently banning contributors

---

## Reporting Issues

If you experience or witness unacceptable behavior, please report it by:

- Opening a private issue (if enabled), or
- Contacting the maintainers directly

All reports will be handled confidentially and reviewed as soon as possible.

---

## Enforcement Guidelines

Maintainers will follow these general guidelines:

1. **Correction** – A private warning for minor issues  
2. **Warning** – A formal warning for repeated behavior  
3. **Temporary Ban** – Suspension from the project  
4. **Permanent Ban** – Removal from all project spaces
